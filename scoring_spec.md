# AI競艇メディア 算出根拠仕様書 (scoring_spec.md)

本ドキュメントは、AI競艇新聞・危険艇速報・万舟警報・転がし候補・激走モーター・
覚醒モーター・AI実績ページ・買い目ロジックで使用する全ての算出項目について、
使用データ・算出方法・更新タイミング・データ区分を明文化したものである。

**運用ルール**：新しい特徴量・スコアリングロジックを追加した場合、本ドキュメントを
同時に更新すること。コードとドキュメントの乖離を防ぐため、実装変更のプルリクエストには
本ファイルの更新を含めることを推奨する。

**朝刊AIポリシーとの関係**：2026年7月の仕様変更（朝刊AI v2.0）により、
previews API由来のデータ（展示タイム・展示ST・風速・風向・波高等）は
全ての予想ロジックから除外されている。これらは「当日リアルタイムデータ」として
本ドキュメント中で明示し、使用箇所がある場合は「教師データとしてのみ」と注記する。

最終更新: 2026-07-03　対象コードバージョン: asahi-v1.0-phase1

## 1. 危険度スコア (danger_score)

────────────────────────
### 使用データ
────────────────────────
- BoatraceOpenAPI programs v2（出走表API。エンドポイント: `https://boatraceopenapi.github.io/programs/v2/{year}/{date}.json`）
- fanファイル（ボートレース公式「モーターボートファン手帳」、月次配布。リポジトリに `fan260X.txt` として保存）

────────────────────────
### 使用カラム
────────────────────────
| カラム | 由来 | 説明 |
|---|---|---|
| `racer_national_top_1_percent` | programs API | 全国勝率 |
| `racer_assigned_motor_top_2_percent` | programs API | モーター2連率 |
| `racer_average_start_timing` | programs API | 平均ST（全国、実績値） |
| `racer_class` | programs API | 級別（A1/A2/B1/B2） |
| コース別ST・進入回数・ST順位 | fanファイル（バイトオフセット82〜、1コース13byte） | 進入コース別の実績ST・ST順位 |

────────────────────────
### 算出方法
────────────────────────
実装: `x_asahi_scoring.calc_danger_score_v2()`
設定: `asahi_config.json` の `danger_score.weights` / `danger_score.thresholds`

100点満点で以下6項目を加算する（Phase1初期重み、業界知見ベース）：

```
danger_score =
    win_rate_low(最大28点)     # 1号艇の全国勝率が他艇平均の85%未満の場合、比率に応じて加点
  + motor_bad(最大22点)        # 1号艇のモーター2連率が他艇平均の90%未満の場合、比率に応じて加点
  + avg_st_slow(最大18点)      # 平均ST >= 0.18 で満額、>= 0.16 で55%
  + class_gap(最大14点)        # 1号艇がB1/B2級の場合。他艇にA1級がいれば満額、いなければ60%
  + course_st_slow(最大10点)   # 1コースST実績 >= 0.18 で満額、>= 0.16 で50%
  + course_rank_bad(最大8点)   # 1コースST順位実績 >= 4.0位で満額、>= 3.0位で50%
```

各重みの採用理由は `asahi_config.json` 内の `reason` フィールドに記載。

────────────────────────
### 更新タイミング
────────────────────────
毎朝1回（cron: UTC 19:15 = JST 04:15）。生成後は当日中固定・再計算しない（朝刊AIポリシー）。

────────────────────────
### 使用可否・データ区分
────────────────────────
| 項目 | 区分 |
|---|---|
| 全国勝率 | 【朝取得データ】 |
| モーター2連率 | 【朝取得データ】 |
| 平均ST（全国実績） | 【朝取得データ】 |
| 級別 | 【朝取得データ】 |
| コース別ST実績（fanファイル） | 【過去実績データ】（前期/後期に1回更新される確定値） |
| 展示タイム・展示ST | 【使用しない】previews API由来のため予想には不使用 |

────────────────────────
### 出力先
────────────────────────
危険艇速報／AI競艇新聞／AI一致指数（構成要素）／買い目ロジック（EVとは独立の危険度表示）／AI実績ページ（達成率算出の母数判定には使わないが、危険艇スコアの内訳として表示）

## 2. 荒れ確率・upset_score

────────────────────────
### 使用データ
────────────────────────
- BoatraceOpenAPI programs v2
- fanファイル
- `VENUE_UPSET_FACTOR`（会場ごとの統計的な荒れやすさ、コード内定数）
- `GRADE_EFFECTS`（レースグレードによる荒れ補正、コード内定数）

────────────────────────
### 使用カラム
────────────────────────
危険度スコアと同じ（全国勝率・モーター2連率・平均ST・級別・コース別ST実績）に加えて、
`race_grade_number`（programs API、レースグレード）、`race_stadium_number`（開催場番号）、
ナイター場フラグ（コード内定数 `NIGHT_VENUES`）。

────────────────────────
### 算出方法
────────────────────────
実装: `x_asahi_scoring.calculate_upset_score_v2()`
設定: `asahi_config.json` の `upset_prob.effects`

1. 各艇の相対スコアを `calc_boat_score_v2()` で算出（平均ST・コース別ST・モーター・勝率・進入枠の重み付き合成、softmaxで確率化）
2. logit/sigmoid変換を用いて、以下の効果量を1号艇の荒れ確率(upset_prob)に順次加算する：

```
upset_prob = 1 - (1号艇の相対確率)

+ st_risk_avg_st(+0.30)       # 1号艇の平均ST実績 > 0.17
+ st_risk_course_st(+0.25)    # 1号艇のコース別ST実績 > 0.16
+ win_rate_risk_1(+0.50)      # 1号艇の全国勝率 < 5.0
+ win_rate_risk_2(+0.20)      # 1号艇の全国勝率 < 5.5（上記と排他）
+ motor_risk(+0.20)           # 1号艇のモーター2連率 < 35.0
+ class_risk(+0.30)           # 1号艇がB1/B2級
+ boat2_strength_mult(×2.0)   # 2号艇の相対確率が1号艇を上回る場合の差分増幅
+ venue_factor                # 会場統計（VENUE_UPSET_FACTOR、-0.15〜+0.15）
+ night_factor(-0.08)         # ナイター場の場合
+ class_gap_effect(+0.40)     # 1号艇B級かつ他にA1級がいる場合
+ race_grade_factor           # GRADE_EFFECTS（一般戦0、G3-0.1、G2-0.2、G1-0.3、SG-0.5）
```

3. `danger_score >= 60` なら upset_score を1.30倍、`>= 40` なら1.15倍に強化
4. 1号艇の相対確率が65%超かつ最有力対抗より高い場合は upset_score = 0（堅い予想と判定）
5. **1号艇等級フィルタ**：1号艇がA1級なら upset_score = 0（強制スキップ）、A2級なら -1.5点

────────────────────────
### 更新タイミング
────────────────────────
毎朝1回。危険度スコアと同時に確定・固定。

────────────────────────
### 使用可否・データ区分
────────────────────────
危険度スコアと同一。会場統計・レースグレードも【朝取得データ】（出走表確定時点で分かる情報）。

────────────────────────
### 出力先
────────────────────────
危険艇速報（1号艇が飛ぶ予想の根拠）／万舟警報（荒れレース抽出の基礎値）／買い目ロジック（EV計算の入力）／note新聞の荒れ指数表示

## 3. AI一致指数 (match_index)

────────────────────────
### 使用データ
────────────────────────
新聞掲載時点の各ブランド掲載状況（`ranking_cache.json` の `all_danger` / `all_manshuu` 等）

────────────────────────
### 使用カラム
────────────────────────
各ブランドの `score`（危険度スコア・荒れ指数）、掲載有無

────────────────────────
### 算出方法
────────────────────────
実装: `x_note_report._calc_match_index()`
設定: `x_brand_config.py` の `MATCH_INDEX_POINTS` / `MATCH_INDEX_RANK_BONUS_S` / `MATCH_INDEX_RANK_BONUS_A`

```
match_index = Σ(掲載ブランドの配点 × ランクボーナス)

配点: danger=20, manshuu=20, korogashi=20, motor_hot=15, motor_awk=15
ランクボーナス: rank_of(score)がSランク(80+)なら×1.15、Aランク(65+)なら×1.05
（rank_ofの閾値は x_brand_config.RANK_THRESHOLDS を単一の基準とする）
```

複数のAIブランドが同じレースを推すほど高スコアになる設計。100点上限でクランプ。
「高配当期待」ブランドは2026年7月の仕様変更で廃止済み、配点対象から除外。

────────────────────────
### 更新タイミング
────────────────────────
毎朝1回、新聞生成と同時。

────────────────────────
### 使用可否・データ区分
────────────────────────
【朝取得データ】（各ブランドスコア自体が朝データのみで算出されているため、間接的に朝データ）

────────────────────────
### 出力先
────────────────────────
AI競艇新聞「📊今日のAIランキング」（掲載順位付け）／AI実績ページ（AIランキング払戻一覧のソートキー）／買い目ロジック（BuyScoreの構成要素の一つ）

## 4. 万舟警報の荒れ指数 (upset_index)

────────────────────────
### 使用データ
────────────────────────
BoatraceOpenAPI programs v2

────────────────────────
### 使用カラム
────────────────────────
`racer_national_top_1_percent`（全国勝率）、`racer_class`（級別）、`racer_average_start_timing`（平均ST）、`racer_assigned_motor_top_2_percent`（モーター2連率）

────────────────────────
### 算出方法
────────────────────────
実装: `x_ranking.calc_upset_index()`

0〜100点、4項目の加重平均（previews由来の展示差・気象リスクを除外した分を朝データ4項目へ比例配分・再正規化済み）：

```
upset_index =
    in_score(重み33.3%)      # 1号艇勝率が低いほど高スコア（100-勝率×10）。B級なら+20
  + close_score(重み26.7%)   # 6艇の勝率標準偏差が小さい（実力接近）ほど高スコア
  + st_var(重み20.0%)        # 6艇の平均ST標準偏差が大きい（ばらつき大）ほど高スコア
  + mg_score(重み20.0%)      # 最大-最小モーター2連率の差（格差）が大きいほど高スコア
```

────────────────────────
### 更新タイミング
────────────────────────
毎朝1回。

────────────────────────
### 使用可否・データ区分
────────────────────────
全項目【朝取得データ】。展示差・気象リスク（旧版で合計25%の重みを占めていた）は完全除外。

────────────────────────
### 出力先
────────────────────────
万舟警報（掲載順位付け）／AI競艇新聞

## 5. 激走モーター指数

────────────────────────
### 使用データ
────────────────────────
`motor_history.csv`（自前で蓄積する、確定済みレース結果のモーター別履歴）

────────────────────────
### 使用カラム
────────────────────────
`place`（着順、確定結果）、`ex_time`（展示タイム、**過去のレースで実際に記録された確定値**）、モーター2連率（programs APIの当日値、比較対象として使用）

────────────────────────
### 算出方法
────────────────────────
実装: `x_ranking.calc_hot_motor()`

直近10走のデータが必要（未満は算出不可＝None）。

```
激走指数 =
    r10(重み35%)       # 直近10走の着順スコア平均（1着=100点〜6着=0点）
  + r20(重み20%)       # 直近20走の着順スコア平均
  + ex_dev(重み25%)    # 直近10走 vs 全履歴平均の展示タイム偏差（速くなっていれば高スコア）
  + gap(重み20%)       # 直近10走の実測2連率 と 当日の公式モーター2連率 の乖離
```

────────────────────────
### 更新タイミング
────────────────────────
`motor_history.csv` は毎朝の危険艇生成時（`--update-history`）に前日分を追記。
激走指数自体も毎朝1回算出・固定。

────────────────────────
### 使用可否・データ区分
────────────────────────
| 項目 | 区分 |
|---|---|
| 直近10走・20走の着順（`motor_history.csv`） | 【過去実績データ】確定済みレース結果 |
| 展示タイム偏差（`motor_history.csv`内の`ex_time`） | 【過去実績データ】※重要：これは**過去に確定した**展示タイムであり、当日これから行われるレースの展示タイム（previews APIのリアルタイムデータ）ではない。当日リアルタイムの展示タイムは一切使用しない。 |
| 当日の公式モーター2連率 | 【朝取得データ】 |

────────────────────────
### 出力先
────────────────────────
激走モーター速報／AI競艇新聞

────────────────────────
### 既知の制約
────────────────────────
`motor_history.csv` のキーは `(venue_num, motor_no)`（会場・モーター番号単位）であり、
レース番号を持たない。そのため「そのモーターがどのレースの何号艇として使われ、
結果として何着だったか」を `hit_record.csv` と直接突合することができず、
**AI実績ページでの的中率算出は現時点で未実装（データ整備中）**。

## 6. 覚醒モーター指数

────────────────────────
### 使用データ
────────────────────────
`motor_history.csv`（激走モーターと同一データソース）

────────────────────────
### 使用カラム
────────────────────────
`place`（着順）、`ex_time`（過去の確定展示タイム）

────────────────────────
### 算出方法
────────────────────────
実装: `x_ranking.calc_awakening()`

直近20走のデータが必要（未満は算出不可＝None）。

```
覚醒指数 =
    trend(重み40%)      # (直近50走平均着順 - 直近10走平均着順)×30+50。短期的な着順改善を検出
  + climb(重み30%)       # (直近10走2連率 - 直近50走2連率)×500+50。勝ち上がり速度
  + ex_improve(重み30%)  # (直近20走展示平均 - 直近5走展示平均)×1000+50。展示タイムの改善度
```

────────────────────────
### 更新タイミング
────────────────────────
激走モーターと同一（毎朝、`motor_history.csv` 更新と同時）。

────────────────────────
### 使用可否・データ区分
────────────────────────
激走モーターと同一。全項目【過去実績データ】、当日リアルタイムpreviews由来データは不使用。

────────────────────────
### 出力先
────────────────────────
覚醒モーター速報（覚醒度90以上のみ配信）／AI競艇新聞

────────────────────────
### 既知の制約
────────────────────────
激走モーターと同様、レース番号との紐付けがなく、**AI実績ページでの的中率算出は現時点で未実装（データ整備中）**。

## 8. BuyScore

────────────────────────
### 使用データ
────────────────────────
`_evaluate_bets()`（notify_arashi.py）が算出する候補ごとのEV・確率・不確実性等、
および本ドキュメント上記1〜3の各スコア。

────────────────────────
### 使用カラム
────────────────────────
| 特徴量 | 由来 |
|---|---|
| `ev` | MLモデル予測確率×オッズから算出される期待値 |
| `prob` | 的中確率（モンテカルロシミュレーション＋MLモデルの合成） |
| `match_index` | 本ドキュメント3節 |
| `market_gap` | AIのモデル内順位と市場（オッズ）順位の乖離 |
| `uncertainty` | モデル予測の不確実性（分散指標） |
| `disagreement` | 複数モデル間の予測の割れ具合 |
| `composite` | キャリブレーション後の信頼度 |
| `race_type` | レースタイプ（本命戦/混戦/超混戦/荒れ戦） |
| `odds` | 3連単オッズ（帯域別に補正） |

────────────────────────
### 算出方法
────────────────────────
実装: `x_buyscore.calc_buyscore()`
設定: `buyscore_config.json` の `weights` / `thresholds` / `odds_band_bonus`

```
BuyScore(0〜100) = 100 × (
    ev_normalized(÷2.0)         × 重み0.35
  + prob_normalized(÷0.08)      × 重み0.20
  + match_index_normalized      × 重み0.15
  + market_gap_normalized       × 重み0.10
  + uncertainty                 × 重み-0.10   # 負の重み（不確実性が高いほど減点）
  + disagreement                × 重み-0.05   # 負の重み
  + composite                   × 重み0.10
)
+ race_type補正（本命戦+0.05×重み0.05、荒れ戦+0.05×重み0.05等）
+ オッズ帯補正（6-25倍で+0.10、80倍超で-0.20等）
+ 展示未取得ペナルティ(-0.08)   # 【朝刊AI】常時適用（展示は使用しないため）
```

BuyScoreが閾値（`buyscore_min=60`）未満、またはEV・AI一致指数・不確実性等の
個別閾値のいずれかに抵触する場合は「見送り」と判定し、買い目を生成しない
（`check_passthrough()`）。

────────────────────────
### 更新タイミング
────────────────────────
15分ごとの荒れ検知実行時に都度算出（買い目候補が変わるため）。
**ただし朝刊AIポリシーにより、実際に配信・記録される「確定予想」は朝1回生成分のみ**。

────────────────────────
### 使用可否・データ区分
────────────────────────
全構成要素が【朝取得データ】または算出済みスコア由来。展示未取得ペナルティは
「展示を使わない」ことそのものへの固定ペナルティであり、previews由来データは不使用。

────────────────────────
### 出力先
────────────────────────
買い目ロジック（◎○▲△ランク付け、最大4点）／学習ログ（`buyscore_log.jsonl`）

## 9. Kelly係数

────────────────────────
### 使用データ
────────────────────────
`prob`（的中確率）、`odds`（オッズ）、`buyscore`（本ドキュメント8節）

────────────────────────
### 使用カラム
────────────────────────
上記3項目のみ

────────────────────────
### 算出方法
────────────────────────
実装: `x_buyscore.kelly_fraction()`
設定: `buyscore_config.json` の `kelly`

```
基本Kelly = (オッズ-1) × 確率 - (1-確率)  を  (オッズ-1) で割った値
フラクショナルKelly = 基本Kelly × fraction(0.25)

BuyScore < 50 → Kelly = 0（見送り相当）
BuyScore < 60 → Kelly × 0.5（縮小）
```

────────────────────────
### 更新タイミング
────────────────────────
BuyScoreと同時。

────────────────────────
### 使用可否・データ区分
────────────────────────
BuyScoreに準ずる（間接的に【朝取得データ】）。

────────────────────────
### 出力先
────────────────────────
買い目ロジック（ベット額の決定）

## 10. AI信頼度（危険艇・万舟）

────────────────────────
### 使用データ
────────────────────────
`hit_record.csv`（過去30日分）

────────────────────────
### 使用カラム
────────────────────────
`upset_score`, `result_combo`, `payout`, `brands`（新方式）

────────────────────────
### 算出方法
────────────────────────
実装: `x_verification.aggregate_by_rank()` / `calc_brand_trust()`

```
1. upset_score(0-9.5) を ÷9.5×100 で0-100スケールに変換
2. rank_of()でS/A/B/Cランク判定
3. brands列があれば、そのレコードが掲載されたブランド（danger/manshuu/hot_high/korogashi）
   ごとに、ランク別の的中判定を集計
   - danger: 実際の1着が1号艇以外なら成功
   - manshuu/hot_high/korogashi: 払戻がそれぞれ10,000円/5,000円/2,700円を超えたら成功
4. brands列がない旧データは upset_score・payout から近似判定（フォールバック）
5. 信頼度ランク = 成功率に応じて A+(65%+)/A(55%+)/B(45%+)/C(0%+)
```

────────────────────────
### 更新タイミング
────────────────────────
AI実績ページ生成時（毎朝、前日分確定後）に過去30日分を再集計。

────────────────────────
### 使用可否・データ区分
────────────────────────
【過去実績データ】（確定済みレース結果の集計）

────────────────────────
### 出力先
────────────────────────
AI実績ページ「AI信頼度」セクション

## 11. AI実績ページ集計（掲載件数ベース成功率）

────────────────────────
### 使用データ
────────────────────────
`daily_stats.json`（新聞生成時に保存される、その日の掲載レース一覧）
`hit_record.csv`（結果照合済みデータ）

────────────────────────
### 使用カラム
────────────────────────
`daily_stats.json`: `danger.races`, `manshuu.races`, `manshuu.top10`, `ranking`（AI一致指数トップ10）
`hit_record.csv`: `venue_num`, `race`, `result_combo`, `payout`

────────────────────────
### 算出方法
────────────────────────
実装: `x_results_page._calc_brand_results()` / `_calc_brand_results_range()` / `_calc_ranking_payouts()`

```
危険艇: 新聞掲載20件のうち、実際に1号艇以外が1着だった件数の割合
万舟:   新聞掲載レースの上位10件のうち、payout>10,000円だった件数の割合
高配当: 新聞掲載レース全件（20件）のうち、payout>5,000円だった件数の割合
中穴:   同上、payout>2,700円

AIランキング払戻一覧: daily_stats.jsonのranking(AI一致指数トップ10)と
                      hit_recordを(venue_num, race)で突合し、各レースの
                      実際の払戻・結果を表示
```

「掲載件数」を分母にすることで、新聞に実際に載った件数を正確な母数として
成功率を計算する（従来の `hit_record.csv` 全体候補ベースとは異なる）。

────────────────────────
### 更新タイミング
────────────────────────
毎朝、危険艇速報と同時（前日分の完全確定結果を対象）。

────────────────────────
### 使用可否・データ区分
────────────────────────
【過去実績データ】（前日分の確定結果）

────────────────────────
### 出力先
────────────────────────
AI実績ページ「本日のAI成績サマリー」「今日のAIランキング払戻一覧」

## 付録: previews API由来データの完全一覧（使用しない項目）

以下は全て「【使用しない】」区分。教師データとして `hit_record.csv` / `sent_*.txt` に
保存されることはあるが、いかなる予想・スコアリング・ランキング・買い目判定にも使用しない。

| 項目 | APIフィールド名 | 教師データとしての保存先 |
|---|---|---|
| 展示タイム | `racer_exhibition_time` | `hit_record.csv`（保存しない、Phase1時点） |
| 展示ST | `racer_start_timing`（previews） | 同上 |
| チルト角 | `racer_tilt_adjustment` | 同上 |
| 風速 | `race_wind` | `hit_record.csv` `wind_speed`列 |
| 風向 | `race_wind_direction_number` | `hit_record.csv` `wind_dir`列 |
| 波高 | `race_wave` | `hit_record.csv` `wave`列 |
| 天候 | `race_weather_number` | 保存しない |

取得自体は `notify_arashi._extract_preview_raw()` で継続し、`_PREVIEW_RAW_CACHE`
経由で教師データ用途にのみ利用する（`build_race_data()` / `generate_all_rankings()`
が組み立てる予想用の `boats` / `weather` オブジェクトには一切反映しない）。
